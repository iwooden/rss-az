Analyze a Rolling Stock Stars self-play game log and produce a structured assessment of model quality.

The game log file path is: $ARGUMENTS

If no path is given, look for `gamelog.md` in the project root.

## Instructions

**Before reading the game log**, read `RULES.md` in the project root. You need to understand the game's rules to assess whether the model's play is strategically sound — e.g., auction mechanics, corporation formation, acquisition pricing, dividend implications, end-game scoring. Do not rely on general knowledge; the rules of Rolling Stock Stars differ from other 18xx/stock games.

Then read the entire game log file. It may be large (5,000-10,000+ lines), so read it in chunks using the Read tool with offset/limit. Read the full file before producing analysis — do not summarize from partial data.

The log is generated by `train/analyze_game.py` using a model checkpoint. Each decision point ("Step N") shows:
- **NN Values**: Raw value head output (per-player, canonical order, tanh range [-1, +1])
- **NN Priors**: Policy head softmax output for legal actions
- **MCTS Visits**: Visit counts after search (1600 simulations typical)
- **Q values**: Mean backed-up value per action from MCTS
- **A0GB Value**: Greedy leaf backup value target (training signal)
- **Action taken**: The action chosen (argmax of visits in analyze mode)

State dumps appear on phase/turn changes showing player cash, net worth, income, companies, corporations, shares, auction row, and deck.

### 2. Value Head Calibration

Assess how well the value head tracks actual outcomes over the course of the game.

- **Early game**: Are all players' values close to each other (~0.6-0.7 range)? This is expected when the outcome is uncertain.
- **Mid/late game**: Do values diverge to reflect the actual standings? The eventual winner's value should trend toward +1, losers toward -1.
- **Value accuracy**: Compare the final NN values and A0GB values at the last few steps to the actual outcome. A well-calibrated model should have the winner near +1 and the last-place player near -1 by the final turns.
- **Value head surprises**: Call out any moments where values shift dramatically between consecutive steps (a sign the model was surprised by an outcome).

### 3. Policy Quality

Assess how well the policy head (NN priors) guides search.

- **Prior-visit agreement**: How often does the NN's top prior match the most-visited MCTS action? High agreement means the policy is already good; low agreement means MCTS is doing heavy correction.
- **Prior concentration**: Track how the entropy of the prior distribution changes over the game. Early game should have more spread (many reasonable actions); late game should be more concentrated (fewer meaningful choices). Extremely flat priors everywhere suggest the policy head hasn't learned much.
- **MCTS override examples**: Highlight 2-3 notable cases where MCTS strongly disagreed with the NN prior (the top prior action got few visits, or a low-prior action dominated visits). These reveal what the model still needs to learn.

### 4. Search Quality

Assess whether MCTS is adding value beyond the raw NN.

- **Visit concentration**: What fraction of visits go to the top action on average? Very high concentration (>80%) means the model is confident. Very low (<20%) means the model is uncertain or the position is genuinely complex.
- **Q-value spread**: How much do Q values differ between the top actions? A spread of <0.01 suggests the model sees many actions as equivalent (possibly undertrained). A spread of >0.1 suggests clear preferences.
- **Terminal vs NN-backed values**: In late-game positions, are the A0GB targets coming from terminal nodes (exact game outcomes) or from NN leaf evaluations?

### 5. Strategic Observations

Comment on the actual gameplay quality, noting signs of learning (or lack thereof). Look for:

- **Auction behavior**: Does the model bid sensibly? Does it overbid for low-value companies or underbid for high-value ones? Does it know when to leave an auction?
- **Company/corporation strategy**: Does the model acquire companies that synergize? Does it form corporations at reasonable times with appropriate par values?
- **Share management**: Does the model buy/sell shares strategically? Does it understand the value of controlling corporations?
- **Income management**: Does the model maintain positive income? Does it close negative-income companies appropriately?
- **End-game awareness**: Does the model behave differently when the end card is near? Does it maximize net worth in the final turns?

For each observation, cite the specific step number(s) so the user can cross-reference.

### 6. Training Assessment

Based on all the above, give a brief overall assessment:
- What has the model learned well so far?
- What are the biggest gaps in its play?
- What would you expect to improve with more training epochs?
- Any red flags suggesting problems with the training setup?

## Output format

Use markdown with the section headers above. Be concise but specific — cite step numbers, exact values, and concrete examples rather than vague generalities. The user is an experienced ML practitioner who understands AlphaZero; don't explain basic concepts.
