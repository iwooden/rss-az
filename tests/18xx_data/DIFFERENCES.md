There are several differences between our game engine and the 18xx game engine. Overall, we should assume that the 18xx is a "known good" implementation of Rolling Stock Stars - it's *very likely* that walking through a complete 18xx game will uncover bugs in our game engine implementation. We must fix those bugs.

That being said, we also need to "translate" the 18xx game actions into something our engine can work with. Here's a (non-exhaustive) list of differences:

* The deck order on 18xx is randomly determined via seed - this is very likely different than our deck setup algorithm, so we can't directly use the seed. Use the game log from the accompanying `.md` file - you can use the initial auction companies and grep for "revealed from deck" to find the order.
* In the INVEST phase, 18xx players can start an auction on any available company (assuming they have the cash). Our game engine optimizes things slightly - the action offsets are set up so that they correspond to the available companies in ascending face-value order. So if we have available companies with face values 3, 5, 7 and 10, if an 18xx player bids on the 7-value company, we need to translate that to make sure it's the third company action offset.
* In the ACQUISITION phase, 18xx players can perform acquisitions in any arbitrary order. In our game engine, we calculate all possible acquisitions, then offer them to the NN in a specific order (for efficiency and NN learning effectiveness). This means that when the ACQUISITION phase starts, we'll need to collect all the acquisitions done by players, and translate those into pass or acquire actions for each acquisition offered to the NN. Note that in 18xx, this means that the Acquisition phase will only end once all players have passed.
* The CLOSING phase is similar to the above - players can close negative-income companies in any arbitrary order, but we calculate all possible company closings and present them to the NN one-by-one. A similar translation will be needed here.
* The 18xx engine is very agressive about automatic actions - any time that a player has 1 possible action (especially "pass"), the 18xx game engine will auto-advance to the next decision point. Our game engine should do the same - we don't want to waste GPU time presenting the NN with "forced" decisions.
* In the 18xx engine, players can set up "auto-pass" - this means that any time the player would be presented with a decision, the engine passes for them. We should keep track of auto-pass state for each player, and execute pass actions where appropriate in order to keep "in sync" with the 18xx actions.

More differences may be found - you should keep track of them here. In short - we should write the test harness to handle translation of 18xx game actions to our engine where we've made a *conscious design choice* to have our game engine differ. In all other cases, we should fix the bugs in our game engine so that it's in sync with the 18xx one. If you are unsure whether some difference is intentional, don't guess - ask me.

When implementing the test, make liberal use of the game log to make sure you're in sync. If you need running totals for anything (corporation cash, player net worth) at a specific point in the game, ask the user.
