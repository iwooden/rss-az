<objective>
Fix player_stride calculation bug in state layout that causes player 1+ and FI data corruption

Purpose: The state layout compute_layout() function is missing is_auction_high_bidder in player_stride calculation. This causes a 1-byte-per-player offset drift:
- Player 0: correct (no stride multiplication)
- Player 1: offset by 1 float (reads/writes wrong memory)
- Player 2: offset by 2 floats
- FI: offset corrupted (fi_offset depends on players_size which is wrong)

The fix is a one-line addition to core/state.pyx line ~73 in compute_layout().

Output: Corrected state.pyx where player_stride includes is_auction_high_bidder field
</objective>
