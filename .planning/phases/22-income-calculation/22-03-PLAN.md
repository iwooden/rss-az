  <name>Task 1: Implement apply_income methods</name>
  <files>entities/corp.pyx, entities/fi.pyx</files>
  <action>
Add apply_income method to Corporation and ForeignInvestor classes:

```cython
cpdef void apply_income(self, GameState state, int income):
    """Apply calculated income to entity cash."""
    self.add_cash(state, income)
```

This is intentionally simple - it just wraps add_cash. The separation exists because:
1. calculate_income computes the value (pure function, testable)
2. apply_income mutates state (side effect, separate concern)

For Player, no new method is needed - Player already has get_income() and add_cash().
Phase 23 will call: player.add_cash(state, player.get_income(state))

Note: Per CONTEXT.md, corporations can go negative here. Phase 23 handles bankruptcy detection.
Players and FI cannot go negative per CLOSING phase guarantees - treat as invariant violation if it happens.
