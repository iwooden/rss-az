## Decisions Made

**1. Raw float* pointer interface instead of GameState**
- **Rationale:** Maximum flexibility - functions can be used with any float array, not just GameState
- **Benefit:** Enables reuse in nogil blocks, temporary buffers, or future refactoring

**2. All functions cdef inline noexcept nogil**
- **Rationale:** Zero call overhead (inline), parallelization support (nogil), no exceptions (noexcept)
- **Benefit:** Performance-critical path - no overhead for function calls or GIL acquisition

**3. Three-function minimal API**
- **Rationale:** Covers all one-hot encoding patterns found in codebase (set value, get value, clear all)
- **Benefit:** Simple, focused API that addresses all existing use cases
