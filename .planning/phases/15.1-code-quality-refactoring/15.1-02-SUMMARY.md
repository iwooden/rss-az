## Decisions Made

**Pattern consistency over backward compatibility:** Changed CORPS to list-by-id for consistency with PLAYERS and COMPANIES, even though it required updating 150+ call sites. This makes the entity pattern uniform across the codebase and eliminates the awkward CORPS[CORP_NAMES[corp_id]] double-indexing pattern.

**Added CORPS_BY_NAME for convenience:** Created separate dict for name-based lookup rather than forcing all code to use list access. This provides best of both worlds - efficient ID access for common case, convenient name access when needed.
