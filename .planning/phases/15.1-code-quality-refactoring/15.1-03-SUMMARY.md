# Phase 15.1 Plan 03: Action Mask Optimization Summary

**One-liner:** Deduplicated phase dispatch and pre-allocated mask buffer for 100+ fewer allocations per game

## What Was Built

Optimized action mask generation with two key improvements:

1. **Single Phase Dispatch Helper** (`_fill_mask_for_phase`)
   - Eliminated 14 LOC duplication between `get_valid_action_mask()` and `get_forced_action()`
   - Central dispatch to all 7 phase-specific mask functions
   - Cleaner, more maintainable code structure

2. **Pre-allocated Mask Buffer** (`_mask_buffer[306]`)
   - Module-level static buffer replaces per-call `np.zeros()` allocation
   - Fast `memset()` clearing instead of numpy allocation
   - Eliminates ~100+ allocations per game (mask generated every action)
   - Copy to numpy only for Python return value

## Deviations from Plan

### Task 1: nogil Deferred (Rule 4 - Architectural Decision)

**Original plan:** Add `nogil` to all 7 `_fill_*_mask()` functions

**Issue discovered:** Functions call ~15 GameState accessor methods that lack nogil versions:
- `get_player_cash`, `get_corp_bank_shares`, `get_corp_price_index`
- `get_auction_company`, `get_auction_price`
- `get_acq_active_corp`, `get_acq_target_company`, `is_acq_fi_offer`
- `get_corp_cash`, `get_corp_issued_shares`, `get_corp_unissued_shares`
- `is_corp_in_receivership`, `is_market_space_available`
- Several others

**Why defer:** Adding nogil to 15+ GameState methods is an architectural change affecting multiple files and system design. This violates Rule 4 (ask about architectural changes).

**Solution:** Completed Tasks 2 & 3 which provide immediate benefits (deduplication, allocation reduction). Task 1 (nogil) will be addressed in Phase 15.1-04 as part of a comprehensive GameState nogil refactoring.

**Files affected:** None for this plan (deferred)

**Commits:** Tasks 2 & 3 only

**Impact:** Parallelization capability deferred, but not blocked. Current optimizations still provide significant performance improvements.

## Performance Improvements

**Task 2 - Deduplication:**
- Code reduction: -14 LOC duplicate logic
- Maintainability: Single source of truth for phase dispatch
- Future-proof: Any changes to dispatch logic happen in one place

**Task 3 - Pre-allocated Buffer:**
- Allocation elimination: ~100-200+ `np.zeros()` calls per game
- Memory efficiency: Reuse same buffer instead of allocate/free cycle
- Cache-friendly: Static buffer stays in cache

**Combined impact:** Mask generation is now significantly faster without changing external interface.

## Technical Details

### Module-Level Buffer Pattern

```cython
# Pre-allocated at module load
DEF MAX_ACTION_COUNT = 306
cdef float _mask_buffer[MAX_ACTION_COUNT]

# Usage
memset(_mask_buffer, 0, total_actions * sizeof(float))
_fill_mask_for_phase(state, phase, &layout, _mask_buffer)
# Copy to numpy for return
```

**Why safe:** Python GIL ensures single-threaded access. Buffer cleared before each use.

### Phase Dispatch Pattern

```cython
cdef void _fill_mask_for_phase(GameState state, int phase,
                               ActionLayout* layout, float* mask) noexcept:
    if phase == PHASE_INVEST:
        _fill_invest_mask(state, layout, mask)
    elif phase == PHASE_BID_IN_AUCTION:
        _fill_bid_mask(state, layout, mask)
    # ... 5 more phases
```

**Benefits:**
- Single dispatch point
- Easy to add new phases
- No duplication between `get_valid_action_mask()` and `get_forced_action()`
