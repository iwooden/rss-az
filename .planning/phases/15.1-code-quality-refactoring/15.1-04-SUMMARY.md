## Technical Implementation

### Before Pattern (repeated 15+ times across files)
```cython
cdef int i
for i in range(GameConstants.NUM_COMPANIES):
    state._data[self._offset + i] = 0.0
if company_id >= 0 and company_id < GameConstants.NUM_COMPANIES:
    state._data[self._offset + company_id] = 1.0
```

### After Pattern (single line)
```cython
set_one_hot(state._data, self._offset, GameConstants.NUM_COMPANIES, company_id)
```

### Helper Functions Used
- `set_one_hot(data, offset, size, value)`: Clear array slice, set one element if valid
- `get_one_hot_index(data, offset, size)`: Scan for set element, return index or -1
- `clear_one_hot(data, offset, size)`: Clear entire array slice

All helpers are `cdef inline noexcept nogil` for zero-overhead inlining at call sites.

## Knowledge Transfer

**Pattern established:** When adding new one-hot encoded fields to entities:
1. Import helpers: `from entities.encoding cimport set_one_hot, get_one_hot_index, clear_one_hot`
2. For setters: Use `set_one_hot(state._data, offset, size, value)`
3. For getters: Use `get_one_hot_index(state._data, offset, size)`
4. For clear: Use `clear_one_hot(state._data, offset, size)`
