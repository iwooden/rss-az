# Milestone v5.1: nogil Optimization

**Status:** SHIPPED 2026-01-28
**Phases:** 20
**Total Plans:** 3

## Overview

The v5.1 milestone closes deferred tech debt from Phase 15.1 by enabling `nogil` on all mask generation functions for true thread-level parallelization. This required extending the low-level pointer-based accessor pattern (established for PlayerOffsets) to corp and turn operations, then updating mask functions to use these consistently.

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- Use inline one-hot scanning loops in turn accessors (not encoding helpers) since they operate on turn pointer not data pointer
- Omit price_index, owned_companies, acquisition_companies from CorpOffsets struct as not needed by mask functions
