# Milestone v2.1: Forced Action Auto-Application

**Status:** SHIPPED 2026-01-23
**Phases:** 7-8
**Total Plans:** 3

## Overview

Implement iterative auto-application of forced actions in GameDriver. When exactly one legal action exists, apply it automatically and continue until 2+ choices are available or the game ends. This ensures the neural network only sees states with real decisions. Two phases: core implementation with history tracking (Phase 7), then test infrastructure updates (Phase 8).

## Milestone Summary

**Key Decisions:**
- Exception-based error signaling for iteration limit and zero actions
- Optional history parameter (None by default for zero overhead)
- Iteration limit hardcoded at 100 (not configurable)
- WRAP_UP stub transitions to GAME_OVER until v3+ implementation
- from_array() requires num_players parameter (layout computation dependency)
