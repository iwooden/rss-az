# Requirements: v2.1 Forced Action Auto-Application

**Defined:** 2026-01-21
**Core Value:** Never present model with 0 or 1 legal actions - auto-apply forced actions until real choice exists

## v2.1 Requirements

Requirements for forced action auto-application. Each maps to roadmap phases.

### Helper Infrastructure

- [x] **HELP-01**: `ForcedActionResult` struct with `count` (int) and `action_idx` (int) fields
- [x] **HELP-02**: `_check_forced_action(state)` cdef function returns ForcedActionResult distinguishing 0, 1, and 2+ legal actions
- [x] **HELP-03**: `_apply_single_action(state, action, history)` cdef function applies one action without auto-advance loop

### Core Loop

- [x] **LOOP-01**: `apply_action()` iteratively auto-applies when exactly 1 legal action exists
- [x] **LOOP-02**: Loop exits when 2+ legal actions available
- [x] **LOOP-03**: Loop exits when game phase is GAME_OVER
- [x] **LOOP-04**: Iteration limit guard (100 max) raises error if exceeded
- [x] **LOOP-05**: Zero legal actions (outside GAME_OVER) raises error

### History Tracking

- [x] **HIST-01**: `apply_action()` accepts optional `history` parameter (list, default None)
- [x] **HIST-02**: When history is not None, append `(state.copy(), action)` before each action applied
- [x] **HIST-03**: History includes both explicit and auto-applied actions
- [x] **HIST-04**: No overhead when history is None (production mode)

### Test Infrastructure

- [x] **TEST-01**: `apply_and_track()` fixture/helper in conftest.py that manages history list
- [x] **TEST-02**: Helper provides access to full action history after apply
- [x] **TEST-03**: Helper allows inspection of intermediate states

### Test Updates

- [x] **TUPD-01**: Categorize existing tests by auto-apply impact
- [x] **TUPD-02**: Update tests that assert on intermediate states to use history helper
- [x] **TUPD-03**: Add tests for forced action chains (multiple auto-applies in sequence)
- [x] **TUPD-04**: Add tests for phase transition during auto-apply (BID -> INVEST)
- [x] **TUPD-05**: Add test for iteration limit guard (defensive)
- [x] **TUPD-06**: Add test for zero legal actions error
